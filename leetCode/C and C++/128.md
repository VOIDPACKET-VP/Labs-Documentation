# Description
- Given an unsorted array of integers `nums`, return _the length of the longest consecutive elements sequence._
- You must write an algorithm that runs in `O(n)` time.

**Example 1:**
**Input:** nums = [100,4,200,1,3,2]
**Output:** 4
**Explanation:** The longest consecutive elements sequence is `[1, 2, 3, 4]`. Therefore its length is 4.

**Example 2:**
**Input:** nums = [0,3,7,2,5,8,4,6,0,1]
**Output:** 9

**Example 3:**
**Input:** nums = [1,0,1,2]
**Output:** 3

# Solution
- C
- USING A HASH SET
```
#define SIZE 40001

typedef struct Node {
    int value;
    struct Node* next;
} Node;

typedef struct {
    Node* Bucket[SIZE];
} HashSet;

int hashFunction(int val) {
    int h = val % SIZE;
    return (h < 0) ? h + SIZE : h;
}

void insert(HashSet* set, int val) {
    int h = hashFunction(val);
    // Check if already exists to avoid duplicate nodes
    Node* curr = set->Bucket[h];
    while (curr) {
        if (curr->value == val) return;
        curr = curr->next;
    }
    Node* newNode = malloc(sizeof(Node));
    newNode->value = val;
    newNode->next = set->Bucket[h];
    set->Bucket[h] = newNode;
}

bool contains(HashSet* set, int val) {
    int h = hashFunction(val);
    Node* curr = set->Bucket[h];
    while (curr) {
        if (curr->value == val) return true;
        curr = curr->next;
    }
    return false;
}

int longestConsecutive(int* nums, int numsSize) {
    if (numsSize == 0) return 0;
    HashSet set;
    for (int i = 0; i < SIZE; i++) set.Bucket[i] = NULL;
    for (int i = 0; i < numsSize; i++) {
        insert(&set, nums[i]);
    }
    int maxCount = 0;
    for (int i = 0; i < numsSize; i++) {
        // Check if nums[i] is the START of a sequence
        if (!contains(&set, nums[i] - 1)) {
            int currentNum = nums[i];
            int currentCount = 1;
            while (contains(&set, currentNum + 1)) {
                currentNum++;
                currentCount++;
            }
            if (currentCount > maxCount) maxCount = currentCount;
        }
    }
    return maxCount;
}
```

- USING SORTING 
```
int compare(const void* a, const void* b) {
    int int_a = *(const int*)a;
    int int_b = *(const int*)b;
    return (int_a > int_b) - (int_a < int_b);
}

int longestConsecutive(int* nums, int numsSize) {
    if (numsSize == 0) return 0;
    qsort(nums, numsSize, sizeof(int), compare);
    int longest = 1;
    int currentStreak = 1;
    for (int i = 1; i < numsSize; i++) {
        if (nums[i] == nums[i - 1]) {
            continue;
        }
        if (nums[i] == nums[i - 1] + 1) {
            currentStreak++;
        } else {
            if (currentStreak > longest) {
                longest = currentStreak;
            }
            currentStreak = 1;
        }
    }
    if (currentStreak > longest) {
        longest = currentStreak;
    }
    return longest;
}
```

# Learned 
### 1. The Pointer Remote (`HashSet* set`)

- **The Concept:** In C, passing a variable directly creates a **copy** (Pass-by-Value). Passing a pointer (`*`) passes the **address** (Pass-by-Reference).
    
- **The "Why":** If you want a function to actually modify your Hash Table (like adding a node), you must give it the address.
    
- **The Tool:** Inside the function, we use the arrow operator `->` (e.g., `set->Bucket`) to "jump" from the address to the actual data.
    

### 2. The Hash Function Organizer

- **The Concept:** A mathematical "translator" that turns any integer (the **Key**) into a valid array index.
    
- **The "Why":** It allows $O(1)$ access. Instead of searching the whole array, you jump directly to the "bucket" where that number belongs.
    
- **The Logic:**
    
    - `% SIZE`: Keeps the index within the array bounds.
        
    - `if (h < 0) h += SIZE`: Fixes C’s negative modulo behavior (indices must be positive).
        

### 3. The "Two-Line" Hook (Linked List Insertion)

When two numbers land in the same bucket (a **collision**), we chain them together using these lines:

1. `newNode->next = set->Bucket[h];`
    
    - **The Handshake:** The new node grabs the hand of whoever is currently at the front of the line.
        
2. `set->Bucket[h] = newNode;`
    
    - **The Update:** You place the new node at the front, making it the new "head" of that bucket.
        

- **Crucial:** You _must_ do it in this order. If you do Line 2 first, you "drop the chain" and lose all previous nodes in that bucket (a **Memory Leak**).
